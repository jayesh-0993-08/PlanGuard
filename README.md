# PlanGuard

**Pre-execution failure detection for LLM-based robot task planning.**

PlanGuard intercepts robot task plans generated by large language models and
classifies them for failure risk *before* execution — catching unsafe or
impossible plans at the text level, without requiring a vision-language model,
physical simulation, or robot hardware.

---

## Why PlanGuard

LLM planners like SayCan and Inner Monologue generate action sequences that
look fluent but fail in execution: they reference objects that don't exist,
violate physical constraints, or mis-order dependent steps. Existing detection
systems (REFLECT, AHA, Guardian) catch failures *after* the robot has already
acted — requiring camera feeds, VLMs, or simulation rollouts.

PlanGuard operates earlier: it screens the plan text itself, before a single
motor moves.

| Approach | Stage | Requires VLM | Requires hardware |
|---|---|---|---|
| REFLECT | Post-execution | Yes | Yes |
| AHA | Post-execution | Yes | Yes |
| Guardian | Pre-execution | Yes | Partial |
| **PlanGuard** | **Pre-execution** | **No** | **No** |

---

## Failure Taxonomy

PlanGuard detects five failure types:

1. **Missing precondition** — action requires a state that hasn't been
   established (e.g. `open drawer` before `pick up knife from drawer`)
2. **Physical impossibility** — action violates real-world physics
   (e.g. `place cup inside closed microwave`)
3. **Affordance violation** — action applied to an object that doesn't
   support it (e.g. `pour liquid into knife`)
4. **Object hallucination** — plan references an object not present in
   the scene
5. **Ordering error** — logically dependent steps are sequenced incorrectly

## Dataset

Generated using [AI2-THOR](https://ai2thor.allenai.org/) simulation:

- **1,500 task plans** across 5 failure categories (300 per class) + safe plans
- Environments: kitchen, living room, bedroom, bathroom scenes
- Labeled using Label Studio
- Format: JSONL — each entry contains `plan_text`, `label`, `failure_type`,
  `scene_id`

Dataset generation scripts in `data/generate/`.

---

## Model

- Base: `distilbert-base-uncased` (HuggingFace)
- Fine-tuned: binary head (safe/unsafe) + 5-class failure type head
- Training: HuggingFace `Trainer` API
- Evaluation metric: F1 (macro)

## Quickstart

```bash
git clone https://github.com/[your-username]/planeguard
cd planeguard
pip install -r requirements.txt

# Run inference on a plan
python model/evaluate.py \
  --plan "pick up the knife, open the drawer, place knife inside"

# Start the REST API
uvicorn api.serve:app --reload
# POST /screen  →  {"plan": "...", "safe": false, "failure_type": "ordering_error"}
```

## UGV Integration

PlanGuard runs as a ROS2 node on the autonomous UGV, screening every plan
generated by the onboard Llama3 planner (via Ollama) before execution.

See `ugv/` for the ROS2 integration node. Full UGV setup documented in
[`ugv/README.md`](ugv/README.md).
